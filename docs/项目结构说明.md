# NEX 业务平台 — 项目结构说明

> 企业级订单与授权管理系统，前后端分离架构。

**标记说明：**
- 🟢 **正常** — 功能完整，代码规范
- 🔴 **冗余** — 与其他文件功能重复，可考虑合并或移除
- 🟡 **占位** — 仅为占位符/桩文件，无实质功能
- 🟠 **半成品** — 部分实现或含硬编码演示数据，未对接真实数据
- ⚠️ **存在问题** — 代码/配置存在不一致或潜在 Bug

---

## 一、整体架构

项目于 2026-04 完成前后端分离重构，由原来的单体架构（Express + React）拆分为：

| 模块 | 路径 | 技术栈 | 说明 |
|------|------|--------|------|
| **前端** | `frontend/` | React 19 + TypeScript + Vite 7 + Tailwind CSS | 纯前端 SPA |
| **后端** | `backend/` | Go 1.22 + Gin + SQLite | RESTful API 服务 |
| **数据库** | `data/app.db` | SQLite (WAL) | 共享数据文件，15 张表 |
| **遗留后端** | `server/` | Express + TypeScript | 原有后端代码（保留供参考） |

### 通信方式

- 开发环境：Vite dev server 通过 `proxy` 将 `/api` 请求转发到 `localhost:3001`
- 生产环境：Go 后端同时托管前端静态文件和 API
