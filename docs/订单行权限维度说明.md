# 订单数据行权限维度说明

## 一、概述

行权限（Row-level Permission）控制用户能看到哪些订单数据行。系统根据**当前用户的角色**上配置的行权限规则，逐条过滤订单列表，只返回满足所有规则的订单。

- **权限载体**：`RoleDefinition`（角色定义）
- **过滤逻辑入口**：`utils/rowPermissionFilter.ts → filterOrdersByRowPermissions()`
- **调用位置**：`contexts/AppContext.tsx` 中的 `filteredOrders`，全局生效
- **多条规则关系**：**AND（全部满足）**—— 必须通过每一条规则才能看到该订单

---

## 二、核心类型定义

### 2.1 权限维度枚举

```typescript
type PermissionDimension =
  | 'departmentId'       // 部门
  | 'productId'          // 产品（已声明，未实现）
  | 'customerIndustry'   // 客户行业（已声明，未实现）
  | 'customerLevel'      // 客户等级（已声明，未实现）
  | 'channelId'          // 渠道ID
  | 'buyerType'          // 买方类型
  | 'industryLine'       // 行业推广类
  | 'province'           // 省份
  | 'directChannelId'    // 直接下级渠道
  | 'orderType'          // 订单类型
  | 'orderSource'        // 订单来源
  | 'orderStatus'        // 订单状态
  | 'salesRep'           // 销售
  | 'businessManager'    // 商务
  | 'creator';           // 制单人
```

### 2.2 行权限规则

```typescript
interface RowPermissionRule {
    id: string;
    resource: 'Order' | 'Customer' | 'Opportunity' | 'Product';
    dimension: PermissionDimension;
    operator: 'equals' | 'contains';
    values: string[];
}
```

### 2.3 角色定义中的行权限

```typescript
interface RoleDefinition {
    id: string;
    name: string;
    baseRowPermission?: 'all' | 'custom';  // 默认 'all' 表示看全部
    rowPermissions?: RowPermissionRule[];   // 具体规则列表
    rowLogic?: Record<string, RowLogicConfig>; // 已定义但未使用
    // ...
}
```

## 三、订单行权限的 10 个维度详解

以下是系统中**已配置且已实现**的订单行权限维度（对应 `permissionConfig.ts → resourceConfig` 中 Order 资源的 `dimensions`）：

### 3.1 人员类维度（3 个）

这 3 个维度共享同一套人员匹配逻辑 `evaluatePersonDimension`，支持以下匹配值：

| 匹配值 | 含义 |
|--------|------|
| `self` | 仅看本人负责/创建的订单 |
| `department` | 看本人所在部门负责/创建的订单 |
| `departmentAndChildren` | 看本人所在部门及所有子部门负责/创建的订单 |
| `__self_dept__` | 同 `department` |
| `__self_dept_children__` | 同 `departmentAndChildren` |

#### 3.1.1 销售（`salesRep`）

| 项目 | 说明 |
|------|------|
| **配置标签** | 销售 |
| **维度 ID** | `salesRep` |
| **订单字段** | `order.salesRepId` |
| **匹配逻辑** | 将 `order.salesRepId` 与当前用户进行人员维度匹配 |
| **示例** | `values: ['self']` → 只看自己是销售负责人的订单 |
| **示例** | `values: ['departmentAndChildren']` → 看本部门及下级部门销售负责的订单 |

#### 3.1.2 商务（`businessManager`）

| 项目 | 说明 |
|------|------|
| **配置标签** | 商务 |
| **维度 ID** | `businessManager` |
| **订单字段** | `order.businessManagerId` |
| **匹配逻辑** | 将 `order.businessManagerId` 与当前用户进行人员维度匹配 |
| **示例** | `values: ['self']` → 只看自己是商务负责人的订单 |

#### 3.1.3 制单人（`creator`）

| 项目 | 说明 |
|------|------|
| **配置标签** | 制单人 |
| **维度 ID** | `creator` |
| **订单字段** | `order.creatorId` |
| **匹配逻辑** | 将 `order.creatorId` 与当前用户进行人员维度匹配 |
| **示例** | `values: ['self']` → 只看自己制单的订单 |

---

### 3.2 组织类维度（1 个）

#### 3.2.1 部门（`departmentId`）

| 项目 | 说明 |
|------|------|
| **配置标签** | 部门 |
| **维度 ID** | `departmentId` |
| **订单字段** | 间接取值——通过 `order.salesRepId` 查用户表得到 `users[salesRepId].departmentId` |
| **匹配逻辑** | 判断销售负责人所属部门是否在 `values` 列表中 |
| **示例** | `values: ['c2-d1', 'c2-d1-r1']` → 只看销售负责人属于指定部门的订单 |

> **注意**：此维度基于**销售负责人的部门**，而非订单本身的字段。

---

### 3.3 业务属性类维度（3 个）

#### 3.3.1 行业推广类（`industryLine`）

| 项目 | 说明 |
|------|------|
| **配置标签** | 行业推广类 |
| **维度 ID** | `industryLine` |
| **订单字段** | `order.industryLine` |
| **匹配逻辑** | 判断订单的行业推广类是否在 `values` 列表中 |
| **数据来源** | 下单时从客户的 `industryLine` 或 `industry` 快照写入 |
| **示例** | `values: ['教育', '政务']` → 只看行业推广类为教育或政务的订单 |

#### 3.3.2 省份（`province`）

| 项目 | 说明 |
|------|------|
| **配置标签** | 省份 |
| **维度 ID** | `province` |
| **订单字段** | `order.province` |
| **匹配逻辑** | 判断订单省份是否在 `values` 列表中 |
| **数据来源** | 下单时从客户的 `province` 或 `region` 快照写入 |
| **示例** | `values: ['广东省', '北京市']` → 只看广东省或北京市的订单 |

#### 3.3.3 直接下级渠道（`directChannelId`）

| 项目 | 说明 |
|------|------|
| **配置标签** | 直接下级渠道 |
| **维度 ID** | `directChannelId` |
| **订单字段** | 当 `order.buyerType === 'Channel'` 时取 `order.buyerId`，否则为空 |
| **匹配逻辑** | 判断渠道订单的买方渠道 ID 是否在 `values` 列表中 |
| **示例** | `values: ['ch001', 'ch002']` → 只看指定渠道下单的订单 |

> **注意**：非渠道类订单（`buyerType` 不为 `Channel`）无法匹配此维度，会被过滤掉。

---

### 3.4 订单状态类维度（3 个）

#### 3.4.1 订单类型（`orderType`）

| 项目 | 说明 |
|------|------|
| **配置标签** | 订单类型 |
| **维度 ID** | `orderType` |
| **订单字段** | 实际取 `order.buyerType`（**非** `order.orderType`） |
| **匹配逻辑** | 判断买方类型是否在 `values` 列表中 |
| **可选值** | `'Customer'`（直签）、`'Channel'`（渠道）、`'SelfDeal'`（自成交）等 |
| **示例** | `values: ['Customer']` → 只看直签客户的订单 |

> **⚠️ 已知差异**：配置 UI 标签为"订单类型"，但代码实现匹配的是 `buyerType`（买方类型），而非订单上的 `orderType` 字段（如"新购订单"、"续费订单"）。

#### 3.4.2 订单来源（`orderSource`）

| 项目 | 说明 |
|------|------|
| **配置标签** | 订单来源 |
| **维度 ID** | `orderSource` |
| **订单字段** | `order.source` |
| **匹配逻辑** | 判断订单来源是否在 `values` 列表中 |
| **可选值** | `'Manual'`（手工录入）、`'Import'`（批量导入）、`'API'`（接口对接）、`'Subscription'`（订阅续费）、`'SelfDeal'`（自成交） |
| **示例** | `values: ['Manual', 'Import']` → 只看手工录入和批量导入的订单 |

#### 3.4.3 订单状态（`orderStatus`）

| 项目 | 说明 |
|------|------|
| **配置标签** | 订单状态 |
| **维度 ID** | `orderStatus` |
| **订单字段** | `order.status` |
| **匹配逻辑** | 判断订单状态是否在 `values` 列表中 |
| **可选值** | `'Draft'`（草稿）、`'PendingApproval'`（待审批）、`'Approved'`（已审批）、`'InDelivery'`（交付中）、`'Delivered'`（已交付）、`'Completed'`（已完成）、`'Cancelled'`（已取消）、`'Rejected'`（已驳回） |
| **示例** | `values: ['Approved', 'InDelivery', 'Delivered']` → 只看已审批及之后状态的订单 |

## 五、过滤流程

```
用户登录 → 获取 currentUser.role
    ↓
查找 RoleDefinition → 检查 baseRowPermission
    ↓
┌─ baseRowPermission === 'all' 且无有效规则 → 返回全部订单
└─ 存在有效规则 → 进入过滤
    ↓
提取 resource === 'Order' 且 values 非空的规则
    ↓
计算当前用户所在部门及所有子部门 ID 集合
    ↓
遍历每条订单，逐条规则评估（AND 关系）
    ↓
全部规则通过 → 订单可见 ✓
任一规则不通过 → 订单不可见 ✗
```

## 六、已声明但未实现的维度

以下维度在 `PermissionDimension` 类型中已声明，但在 `evaluateOrderRule` 的 `switch` 中走到 `default: return true`，等价于不过滤：

| 维度 ID | 说明 |
|---------|------|
| `productId` | 按产品过滤（订单含多产品行，需确定匹配策略） |
| `customerIndustry` | 按客户行业过滤（与 `industryLine` 功能重叠） |
| `customerLevel` | 按客户等级过滤 |

## 七、列权限（补充）

除行权限外，系统还支持**列权限**（Column-level Permission），控制用户能看到订单的哪些字段：

| 列 ID | 列标签 |
|--------|--------|
| `total` | 订单金额 |
| `customerName` | 客户名称 |
| `licensee` | 被授权方 |
| `status` | 订单状态 |
| `paymentRecord` | 收款记录 |
| `approval` | 审批记录 |

列权限通过 `RoleDefinition.columnPermissions` 配置，`allowedColumns` 列表决定哪些列可见。

## 八、配置入口

行权限的配置通过以下功能权限控制：

| 权限点 ID | 标签 | 说明 |
|-----------|------|------|
| `role_perm_functional` | 配置功能权限 | 配置角色的功能权限树 |
| `role_perm_row` | 配置行权限 | 配置角色的行级数据权限规则 |
| `role_perm_column` | 配置列权限 | 配置角色的列级数据权限规则 |

配置界面位于：**系统管理 → 用户管理 → 角色权限配置**
