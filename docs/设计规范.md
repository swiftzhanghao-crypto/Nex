# WPS365 业务平台 · 设计规范

> 本文档基于现有代码提取整理，以订单管理模块为核心蓝本，覆盖全局设计体系与交互规范。
> 最后更新：2026-04-16

---

## 一、设计体系总览

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| **一致性** | 全局使用 `unified-*` 组件类，所有页面共享相同的卡片、按钮、标签风格 |
| **层级清晰** | 通过背景色（画布→表面→悬浮）、阴影、圆角三维度区分内容层级 |
| **状态即色彩** | 每种业务状态对应固定色系，颜色即含义，无需额外学习 |
| **渐进披露** | 信息按需展开——搜索→高级筛选→列配置→行详情，减少认知负荷 |
| **暗黑优先适配** | 所有组件均有 `dark:` 变体，通过 `class` 策略切换，跟随系统或手动切换 |

| 用途 | 浅色模式 | 深色模式 | 使用场景 |

## 三、字体与排版

### 3.1 字体族

```css
font-family: -apple-system, BlinkMacSystemFont, 'SF Pro Text', Segoe UI, Roboto, Helvetica, Arial, sans-serif;
```

全局开启 `-webkit-font-smoothing: antialiased` + `-moz-osx-font-smoothing: grayscale`。

### 3.2 字号层级

| 用途 | 类名 | 实际大小 | 字重 | 订单中的应用 |
|------|------|---------|------|-------------|
| 页面标题 | `text-2xl` / `text-xl` | 24/20px | `font-bold` | "订单管理"、"产品目录" |
| 区块标题 | `text-lg` / `text-base` | 18/16px | `font-bold` | 详情页分区标题 |
| 正文 | `text-sm` | 14px | `font-medium` | 表格单元格、表单标签 |
| 辅助文字 | `text-xs` | 12px | — | 筛选器标签、时间戳 |
| 极小文字 | `text-[10px]`/`text-[9px]` | 10/9px | `font-bold` | 状态卡片角标、权限标签 |

| 场景 | 类名 | 说明 |

| 金额 | `font-mono` + `tabular-nums` | 等宽数字对齐，如 `¥12,500.00` |

## 六、组件规范

### 6.1 统一卡片 `.unified-card`

```css
bg-white dark:bg-[#1C1C1E]
rounded-[24px]
shadow-apple
border border-gray-100/50 dark:border-white/10
overflow-hidden
transition-all duration-300
```

所有主要内容容器的基础样式，包括表格容器、筛选面板、弹窗内容区。

### 6.2 统一表头 `.unified-table-header`

```css
bg-gray-50/80 dark:bg-white/5
text-gray-500 dark:text-gray-400
text-sm uppercase font-bold tracking-wider
border-b border-gray-100 dark:border-white/10
```

订单列表、客户列表等所有表格的 `<thead>` 统一样式。

### 6.3 按钮体系

| 类型 | 类名 | 浅色效果 | 深色效果 | 应用场景 |
|------|------|----------|----------|----------|
| 主按钮 | `.unified-button-primary` | 蓝底白字 `#0071E3` | 红底白字 `#FF2D55` | 提交、确认、创建订单 |
| 次按钮 | `.unified-button-secondary` | 白底灰字+边框 | `#1C1C1E` + 灰字 | 取消、返回、筛选 |
| 幽灵按钮 | `.unified-button-ghost` | 透明+悬停变色 | 同 | 工具栏操作 |

**按钮基础** `.unified-button`：

```css
px-4 py-2 rounded-lg font-bold text-sm
transition-all duration-200
active:scale-[0.98]
disabled:opacity-50 disabled:pointer-events-none
flex items-center justify-center gap-2
```

### 6.4 状态标签 `.unified-tag-*`

标签用于表格列、卡片角标、详情页状态展示：

| 变体 | 用途示例 |
|------|---------|
| `unified-tag-blue` | 待审批、待确认、待支付 |
| `unified-tag-green` | 已完成、已支付、在架 |
| `unified-tag-orange` | 备货中、进行中 |
| `unified-tag-red` | 已退款、错误 |
| `unified-tag-indigo` | 已发货 |
| `unified-tag-gray` | 草稿、已取消（虚线边框 `!border-dashed`） |

胶囊形态：追加 `!rounded-full` 使标签变为胶囊样式，用于订单状态列。

### 6.5 图标

- 统一使用 `lucide-react` 图标库
- 默认尺寸：`w-5 h-5`（20px）
- 小图标：`w-4 h-4`（16px）
- 超小图标：`w-3.5 h-3.5`（14px）
- 按钮内图标与文字 `gap-2`

## 七、订单管理页面规范

> 以下基于 `OrderManager.tsx`、`OrderDetails.tsx`、`OrderCreateWizard.tsx` 提炼。

### 7.1 订单列表页（OrderManager）

#### 页面结构

```
┌─────────────────────────────────────────────────┐
│  标题栏：标题 + 搜索 + 工具栏按钮               │
├─────────────────────────────────────────────────┤
│  状态筛选卡片（横向可滚动 snap）                  │
├─────────────────────────────────────────────────┤
│  表格区域（双层同步横向滚动 + 纵向可滚动）        │
│  ┌──────┬─────────────────────────────┬──────┐  │
│  │ 勾选 │ 订单号 │ 客户 │ 状态 │ ... │ 操作 │  │
│  │sticky│ sticky │      │      │     │sticky│  │
│  └──────┴─────────────────────────────┴──────┘  │
├─────────────────────────────────────────────────┤
│  分页栏：页码 + 每页条数切换                      │
└─────────────────────────────────────────────────┘
┌ 批量操作悬浮条（fixed bottom-8, 选中时出现）─────┐
│  已选 N 项  │  批量确认  │  批量发货              │
└─────────────────────────────────────────────────┘
```

#### 状态筛选卡片

- 组件：`StatusFilterCard`
- 布局：`flex overflow-x-auto gap-2 snap-x snap-mandatory`
- 每张卡片：`min-w-[100px]`、`rounded-xl`、`p-2`、`snap-start`
- 三种变体：
  - `primary`（蓝）：默认流水线状态
  - `danger`（红）：退款中、已退款
  - `muted`（灰）：已取消
- 激活态：实色背景 + 白字 + `shadow-lg shadow-{color}-500/20` + 底部白点指示条
- 非激活态：白色背景 + 灰文字 + 悬停 `scale-105`
- 权限控制：`order_view_all` 控制"全部"卡片显示；每个状态有独立 `permission`

#### 搜索与高级筛选

**搜索栏**：
- 高度 `h-9`，圆角 `rounded-lg`
- 左侧字段切换下拉（订单号/客户名/产品名/授权方）
- 聚焦时：`focus-within:border-blue-400` + 蓝色光环
- 清除按钮：输入内容时显示 `X` 图标

**高级筛选抽屉**：
- 触发：工具栏 `Filter` 按钮，需 `order_advanced_filter` 权限
- 容器：右侧抽屉 `max-w-[480px]`，通过 `ModalPortal` 渲染
- 条件行：字段（120px）/ 模式（64px）/ 值（flex-1）三列布局
- 支持的模式：单选、多选、时间段、时间点、金额范围
- 交互：编辑完成后点"查询"才生效（`appliedFilters` 与 `advancedFilters` 分离）

#### 表格设计

- 列宽：通过 `colWidthMap` + `<colgroup>` 固定，`tableLayout: 'fixed'`
- 横向滚动：表头和表体两个独立滚动容器，`onScroll` 双向同步 `scrollLeft`
- 纵向滚动：表体 `max-h-[calc(100vh-370px)]` + `custom-scrollbar`
- 固定列：勾选列、订单号列、操作列使用 `sticky` + `z-index` 分层
- 选中行高亮：`bg-blue-50/80 dark:bg-blue-900/10`
- 列配置：需 `order_column_config` 权限；`FIXED_COLUMNS = ['id', 'action']` 不可隐藏
- 列权限：由角色 `columnPermissions` 中 `resource === 'Order'` 的 `allowedColumns` 控制

#### 批量操作

- 触发条件：`selectedOrderIds.size > 0` 时显示底部悬浮条
- 悬浮条：`fixed bottom-8`，`unified-card` + `rounded-full` + `backdrop-blur-xl`
- 两个操作按钮：
  - 批量确认（橙色）：筛选 `PENDING_CONFIRM` 状态订单，需 `order_workflow_confirm`
  - 批量发货（蓝色）：筛选 `PROCESSING_PROD` 状态订单，需 `order_workflow_shipping`
- 全选仅作用于当前页
- 操作前 `confirm()` 确认弹窗

#### 分页

- 默认 `itemsPerPage = 20`，可选 20/50/100
- 底栏：`border-t` + `bg-gray-50/30 dark:bg-white/5`
- 页码文案：`text-xs`
- 翻页按钮：小型 `unified-card` 样式

#### 粘性顶栏

- 背景：`bg-white/90 dark:bg-[#1C1C1E]/90 backdrop-blur-xl`
- 边框：`border-b border-gray-200/60 dark:border-white/10`
- 标题：`text-lg md:text-xl font-bold`
- 返回按钮：`rounded-full` + `hover:bg-gray-100`
- 状态标签：使用 `unified-tag-*` + `!rounded-full`

#### Tab 导航

- 三个 Tab：管理 `MANAGEMENT` / 履约 `FULFILLMENT` / 邮件 `EMAIL`
- 选中态：`text-[#0071E3] dark:text-[#0A84FF]` + 底部蓝色 `border-b-2`
- 权限控制：履约需 `order_detail_delivery`，邮件需 `order_detail_shipping`

#### 流程步骤条

- 容器：`unified-card` + `min-w-[700px]` 横向可滚动
- 步骤节点状态：
  - 已完成：绿色实心圆 `bg-green-500` + `Check` 图标
  - 当前步骤：蓝色实心圆 `bg-[#0071E3]` + `animate-pulse` + "点击处理" 文字
  - 未到达：灰色空心圆 `border-gray-300`
- 步骤连线：已完成 `bg-green-400`，未完成 `bg-gray-200`
- 权限：每个步骤绑定 `stepPermissionMap` 中的功能权限
- 自成交订单（`SelfDeal`）简化为仅支付和交易完成两步

#### 操作抽屉

- 触发：点击当前步骤节点
- 容器：`ModalPortal` + 右侧 `max-w-3xl`
- 动画：`animate-drawer-enter/exit` + `animate-backdrop-enter/exit`
- 关闭时序：先播放退出动画 280ms，再卸载组件

**审批抽屉内容**：
- 黄色提示条 `rounded-2xl`
- 三个审批卡片（销售/商务/财务），未轮到的 `opacity-50` + `Lock` 图标
- 通过：`unified-button-primary`
- 拒绝：红底小按钮

**支付/发货/退款抽屉**：
- 大色块 CTA 按钮：`bg-gradient-to-br from-blue-500 to-blue-600`
- 禁用态：`disabled:opacity-30`
- 表单：标准输入框 + `rounded-xl` + `focus:ring`

### 7.3 新建订单向导（OrderCreateWizard）

#### 整体结构

- 全屏右侧抽屉：`w-[calc(100vw-240px)]`
- 遮罩：`bg-black/40 backdrop-blur-sm`
- `z-[500]`，通过 `ModalPortal` 渲染

#### 步骤条

```
 ① 订单类型  →  ② 客户信息  →  ③ 产品明细  →  ④ 交付信息
  [buyerType]    [customer]     [products]      [delivery]
```

- 轨道：灰色底色 + 蓝色进度条 `transition-all duration-500 ease-out`
- 圆点：完成 `bg-[#0071E3]` + `Check`，当前蓝色实心，未到灰色空心
- 每步标签：`text-xs`，当前步高亮

#### 各步内容

| 步骤 | 内容 | 校验规则 |
|------|------|----------|
| ① 订单类型 | 四宫格选择 `buyerType`（直签/渠道/自成交/兑换码），按类型过滤订单来源 | 必选 `buyerType` |
| ② 客户信息 | 客户选择器、商机关联、渠道选择、联系人选择 | 直签需客户+商机；渠道需渠道 ID；自成交需企业 ID |
| ③ 产品明细 | 产品搜索+添加、SKU/定价选项、数量/单价/授权方 | 至少一个产品项 |
| ④ 交付信息 | 发票、支付方式、收货地址、验收配置 | 企业/客户/产品非空；序列号 20 位；分期验收比例合计 100% |

#### 底栏

- `unified-card` + `shadow-[0_-10px_40px_rgba(0,0,0,.05)]`
- 左侧：上一步（次按钮）
- 右侧：下一步 `bg-[#0071E3]` / 提交 `unified-button-primary` + `hover:scale-105`
- 提交按钮文案：根据 API 模式切换"提交订单"/"暂存到本地"

## 八、弹窗与抽屉规范

### 8.1 遮罩层

- 定位：`fixed inset-0`
- 背景：`bg-black/40` ~ `bg-black/60`
- 模糊：`backdrop-blur-sm`
- 层级：`z-[500]`（标准），`z-[600]`（二级弹窗如员工卡片）
- 渲染：通过 `ModalPortal` 渲染到 `#modal-root`，避免 `overflow: hidden` 裁剪

### 8.2 居中弹窗

- 布局：`flex items-center justify-center`
- 内边距：`p-4` ~ `p-6`
- 圆角：`rounded-3xl`（桌面端）
- 最大尺寸：`max-w-5xl`、`max-h-[90vh]`
- 动画：`animate-modal-enter` 0.4s

### 8.3 侧边抽屉

- 位置：右侧滑入 `justify-end`
- 宽度：`max-w-md`（用户详情）~ `max-w-3xl`（订单操作）
- 进入动画：`animate-drawer-enter` 0.38s `cubic-bezier(0.22,1,0.36,1)`
- 退出动画：`animate-drawer-exit` 0.26s `cubic-bezier(0.4,0,1,1)`
- 遮罩动画：`animate-backdrop-enter` 0.28s / `animate-backdrop-exit` 0.22s
- 关闭模式：先设 `isClosing = true` 播放退出动画 → 280ms 后卸载组件

### 8.4 动画定义总览

| 动画名 | 时长 | 曲线 | 用途 |
|--------|------|------|------|
| `animate-page-enter` | 0.2s | ease-out | 页面切换 |
| `animate-fade-in` | 0.5s | ease-out | 通用淡入 |
| `animate-modal-enter` | 0.4s | `cubic-bezier(0.16,1,0.3,1)` | 弹窗进入 |
| `animate-drawer-enter` | 0.38s | `cubic-bezier(0.22,1,0.36,1)` | 抽屉进入 |
| `animate-drawer-exit` | 0.26s | `cubic-bezier(0.4,0,1,1)` | 抽屉退出 |
| `animate-backdrop-enter` | 0.28s | ease-out | 遮罩进入 |
| `animate-backdrop-exit` | 0.22s | ease-in | 遮罩退出 |

## 九、交互逻辑规范

### 9.1 状态流转交互

订单状态遵循以下流水线，每个流转节点都有对应的操作权限：

```
草稿 → 待审批 → 待确认 → 备货中 → 待支付 → 已发货 → 已完成
 │                                                    ↑
 │  ┌─── 异常分支 ────────────────────────────────────┘
 │  退款中 → 已退款
 └─ 已取消
```

| 流转动作 | 权限 | 触发方式 |
|----------|------|----------|
| 提交草稿 | `order_create` | 向导提交 |
| 审批通过/拒绝 | `order_workflow_approval` | 详情页审批抽屉 |
| 商务确认 | `order_workflow_confirm` | 详情页 / 批量操作 |
| 发货 | `order_workflow_shipping` | 详情页 / 批量操作 |
| 支付确认 | `order_workflow_payment` | 详情页支付抽屉 |
| 申请退款 | `order_refund` | 详情页退款按钮 |

### 9.2 权限驱动的 UI 隐藏

系统采用 RBAC 模型，UI 元素的可见性由当前用户角色权限决定：

| 层级 | 控制对象 | 说明 |
|------|----------|------|
| 功能权限 | 按钮、Tab、菜单项 | `hasPermission('xxx')` 控制是否渲染 |
| 行权限 | 订单列表行 | `rowPermissions` 按部门/区域/来源等维度过滤可见订单 |
| 列权限 | 表格列 | `columnPermissions` 控制哪些列对当前角色可见 |
| 路由守卫 | 页面 | `RequireAnyPermission` 包裹路由，无权限重定向首页 |

### 9.3 搜索与筛选交互

1. **即时搜索**：搜索框输入即触发 `useMemo` 重新计算，无延迟/防抖
2. **状态筛选**：点击状态卡片即时切换 `filterStatus`
3. **高级筛选延迟生效**：编辑筛选条件后，必须点击"查询"才应用到 `appliedFilters`
4. **筛选条件持久化**：切换状态不清除搜索词和高级筛选
5. **分页重置**：筛选条件变化时自动回到第一页

### 9.4 表格交互

1. **行点击**：点击订单行导航到详情页 `navigate('/orders/${id}')`
2. **勾选**：独立于行点击，通过 checkbox 列操作
3. **全选**：仅选中当前页所有订单
4. **复制表格**：`Ctrl+C` 风格按钮将当前筛选结果复制为 TSV 格式
5. **列拖拽排序**：`ColumnConfigModal` 中拖拽调整列顺序
6. **固定列**：`id`（订单号）和 `action`（操作）列始终固定，不可隐藏或移动

### 9.5 向导交互

1. **步骤线性**：必须按顺序完成，不可跳步
2. **步骤校验**：每步的"下一步"按钮在校验不通过时 `disabled`
3. **数据保留**：返回上一步不清除已填写数据
4. **草稿恢复**：可从草稿列表恢复之前保存的未完成订单
5. **续费初始化**：从客户详情页发起续费时，自动填充原订单数据
6. **退出确认**：关闭向导时若已有填写数据，弹出确认对话框

### 9.6 详情页交互

1. **区块折叠**：各信息区块支持 `toggleSection` 折叠/展开
2. **操作抽屉**：点击流程步骤条当前节点打开对应操作抽屉
3. **审批依赖**：三级审批（销售→商务→财务）必须按序完成，未轮到的显示锁定态
4. **乐观更新**：本地模式下操作立即更新 UI；API 模式下等待接口响应后刷新
5. **操作日志**：每次状态变更自动追加 `approvalRecords`，可通过日志抽屉查看完整历史
