# WPS 365 业务平台 — 项目结构说明

> 企业级订单与授权管理系统，集成智能商业洞察。

**标记说明：**
- 🔴 **冗余** — 与其他文件功能重复，可考虑合并或移除
- 🟡 **占位** — 仅为占位符/桩文件，无实质功能
- 🟠 **半成品** — 部分实现或含大量硬编码演示数据，未对接真实数据
- ⚠️ **存在问题** — 代码/配置存在不一致或潜在 Bug

## 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| **前端框架** | React + TypeScript | React 19, TS 5.8 |
| **构建工具** | Vite | 7.x |
| **路由** | React Router DOM (HashRouter) | 7.x |
| **样式** | Tailwind CSS (CDN + 构建工具链) | 3.x |
| **图表** | Recharts | 3.x |
| **图标** | Lucide React | 0.554 |
| **虚拟列表** | @tanstack/react-virtual | 3.x |
| **Markdown 渲染** | react-markdown | 10.x |
| **AI 集成** | @google/genai (Gemini) | 1.x |
| **数据校验** | Zod | 4.x |
| **后端** | Express + better-sqlite3 | Express 5, SQLite |
| **身份认证** | jsonwebtoken (JWT) | 9.x |
| **测试** | Playwright | 1.x |
| **开发工具** | tsx, concurrently | — |

## 脚本命令

```bash
npm run dev           # 启动 Vite 开发服务器（仅前端）
npm run dev:server    # 启动 Express API 服务器
npm run dev:all       # 同时启动前后端
npm run build         # 构建生产版本
npm run preview       # 预览生产构建
npm run lint          # TypeScript 类型检查
npm run db:seed       # 初始化/填充数据库
```

## 架构要点

1. **前后端分离** — 前端 Vite + React SPA，后端 Express REST API，通过 `concurrently` 同时启动开发
2. **双数据源** — `AppContext` 通过 `VITE_API_MODE` 切换 mock 数据（`data/generators.ts`）和后端 API（`services/api.ts`），但切换不完整（绩效/授权/交付信息始终走 mock）
3. **HashRouter** — 使用 Hash 路由，适配静态部署场景
4. **SQLite** — 轻量数据库，通过 `better-sqlite3` 同步访问，无需独立数据库服务
5. **AI 集成** — 通过 `@google/genai` 接入 Gemini，用于产品描述生成和商业洞察分析
6. **权限体系** — `usePermission` Hook + `permissionConfig.ts` 实现前端权限控制，后端 `requireRole` 中间件做角色门控
7. **路径别名** — `@` 映射到项目根目录，在 `vite.config.ts` 和 `tsconfig.json` 中统一配置
