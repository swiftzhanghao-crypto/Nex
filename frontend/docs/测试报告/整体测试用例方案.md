## 一、测试范围与策略

### 1.1 系统模块总览

| 编号 | 模块 | 子模块 | 优先级 | 测试状态 |
|------|------|--------|--------|----------|
| M01 | 数据看板 | Dashboard | 中 | 待测 |

| M03 | 产品中心 | 产品目录/产品管理/销售商品/组件池/安装包/授权模板/属性配置/报价 | 中 | 待测 |

| M08 | 业绩中心 | 业绩列表 | 中 | 待测 |
| M09 | 项目管理 | 项目列表/项目详情 | 低 | 待测 |
| M10 | 系统配置 | 用户管理/角色管理/组织架构/权限控制 | **高** | 待测 |
| M11 | 全局功能 | 布局/导航/主题/登录/路由守卫 | 中 | 待测 |

### 1.2 测试策略

- **功能测试**：覆盖每个模块的 CRUD、业务流程、边界条件
- **权限测试**：验证不同角色（Admin / Sales / Business / Technical / Logistics）的可见性与操作权限
- **工作流测试**：覆盖订单全生命周期（创建 → 审批 → 备货 → 发货 → 验收）
- **数据联动测试**：跨模块数据一致性（订单↔客户、订单↔产品、订单↔合同）
- **异常与边界测试**：必填校验、极值输入、并发操作

### 1.3 测试角色矩阵

| 角色 | 权限级别 | 主要测试场景 |
|------|---------|-------------|
| 超级管理员 (Admin) | `all` | 全功能验证 |
| 销售 (Sales) | 订单创建/客户/商机 | 业务流程、数据权限 |
| 商务经理 (Business) | 审批/合同/发票 | 审批流、财务管理 |
| 技术支持 (Technical) | 备货/授权码生成 | 备货流程、授权管理 |
| 物流专员 (Logistics) | 发货/快递 | 发货流程 |
