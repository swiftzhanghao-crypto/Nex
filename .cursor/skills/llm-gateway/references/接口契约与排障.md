# 接口契约与排障

## API 契约（最小）

推荐请求结构：
- `prompt`: string，必填
- `data`: JSON，必填
- `model`: string，可选，覆盖默认模型

推荐响应结构：
- `content`: string，模型原始文本输出
- `result`: any，可选，解析出的 JSON 结构

对应示例：
- `examples/llm_api_controller_example.go`

## Gateway 请求契约

目标接口：
- `POST {baseUrl}{chatApi}`

关键 Header：
- `Content-Type: application/json`
- `Authorization: Bearer <apiKey>`
- 可选：`AI-Gateway-Uid`、`AI-Gateway-Product-Name`、`Ai-Gateway-Intention-Code`

关键 body 字段：
- `model`
- `messages`
- `stream`
- `temperature`
- `top_p`
- `max_completion_tokens`
- `retry_strategy`

## 配置契约

建议统一在 `llm.gateway.*`：
- `baseUrl`
- `chatApi`
- `modelId`
- `apiKey`
- `temperature`
- `topP`
- `maxCompletionTokens`
- `stream`
- `retryCount`
- `timeoutSec`
- `uid`
- `productName`
- `intentionCode`

配置示例：
- `examples/config.llm.example.yaml`

## 常见故障与处理

- 401 或 403：
  - 检查 `apiKey` 是否有效，是否需要网关白名单。
  - 检查 Authorization 是否是 Bearer 格式。

- 404 或 405：
  - 检查 `baseUrl + chatApi` 拼接是否正确。
  - 检查网关版本路径是否变更。

- 超时或间歇失败：
  - 增加 `timeoutSec`，并配置有限重试。
  - 降低 `maxCompletionTokens`，减少响应体积。

- result 解析为空：
  - 检查 prompt 是否明确要求只输出 JSON。
  - 检查模型输出是否包在 markdown 代码块中。
  - 保留 `content` 供业务降级处理，不要丢弃。

## 安全建议

- 生产环境禁止把真实 `apiKey` 写进仓库配置。
- 错误日志可打印响应摘要，但需做长度截断和敏感字段脱敏。
