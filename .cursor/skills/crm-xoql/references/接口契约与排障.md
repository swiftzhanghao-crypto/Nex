# 接口契约与排障

## Token 接口契约（password 模式）

来源：`docs/xsy/获取token.md` 与现网实现经验。

关键信息：
- 方法：POST
- Content-Type：application/x-www-form-urlencoded
- 典型参数：`grant_type=password`、`client_id`、`client_secret`、`username`、`password`
- 成功返回：包含 `access_token`、`token_type`、`expires_in`

工程建议：
- 生产环境密钥使用环境变量注入，不写入仓库。
- 缓存 TTL 建议短于平台过期时间，例如 23h。
- 空 token 直接判错，不允许回传业务层。

## XOQL 接口契约

来源：`docs/xsy/接口文档.md` 与 `docs/xsy/xoql/`。

关键信息：
- 路径：`/rest/data/v2.0/query/xoql`
- 方法：POST
- Content-Type：application/x-www-form-urlencoded
- Header：`Authorization: Bearer <access_token>`
- 可选 Header：`xsy-criteria: 10`（管理口径）
- 表单参数：`xoql=<查询语句>`

工程建议：
- 统一在服务层规范化 Authorization，避免重复拼接 Bearer。
- 统一兼容响应 `code` 的字符串/数字两种返回形式。
- 对空 SQL、非法 SQL、超长 SQL 做前置校验。
- 生成 SQL 时必须遵守：`references/XOQL生成SQL规范.md`。

## 分页契约

来源：XOQL 平台单次返回上限约 3000 的限制说明。

建议策略：
- pageSize 默认 3000，最大不超过 3000。
- 在基础 SQL 末尾追加 `limit offset,size` 轮询拉取。
- 当单页返回条数小于 pageSize 时停止拉取。
- 聚合后回填 `data.records` 与 `data.count`。

## Demo 接口安全约束

`risk_board` 相关代码在此技能中仅做 SQL 执行演示，不携带业务看板逻辑。建议加以下限制：
- 仅允许 `select` 开头的只读语句。
- 拒绝 `update`、`insert`、`delete`、`drop`、`alter`、`truncate` 等关键字。
- 限制 SQL 长度（如 2000）。
- 接口仅用于联调或内部工具，不直接对外网开放。

## 常见问题排查

- token 获取失败：
  - 检查 `oauthBaseUrl`、`tokenPath` 是否完整。
  - 检查账号密码是否需要拼接动态令牌。
  - 检查 client_id / client_secret 是否与租户一致。

- XOQL 返回非 200：
  - 打印并核对请求头是否带 Bearer。
  - 核对 SQL 字段名是否存在、拼写是否与对象 API 一致。
  - 确认是否需要 `xsy-criteria: 10`。

- 数据量异常：
  - 检查是否遗漏分页聚合。
  - 检查 where 条件时间单位是否为毫秒时间戳。
