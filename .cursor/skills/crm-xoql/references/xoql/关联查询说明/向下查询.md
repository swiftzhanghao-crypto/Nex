向下查询指通过父到子的关系查询数据，用于一对多的关系中，可以通过子查询指定返回符合条件的子对象数据。

在向下查询时，可以在 SELECT、FROM 和 WHERE 子句中直接使用点号（.）表示关联对象上的字段（对象 API 名称.字段 API 名称）。

在向下查询时，使用带括号的子查询。子查询中 FROM 语句中的对象与外部查询（括号外）FROM 语句中的对象具有关联关系。

例如，查询客户对象上的所有客户名称以及每个客户下所有联系人的姓名。

```plain
SQL

SELECT accountName, 
    (SELECT contactName 
     FROM contact 
     WHERE accountId = account.id)
FROM account
```

示例中的 account.id 为关联字段的固定写法，在示例中的含义为联系人对象上的 accountId 字段关联到客户对象上的 id 字段。

+ 向下查询时，最多允许包含 5 个子查询，即最多包含 5 对括号。
+ 向下查询最大层数为 1 层。
+ 向下查询仅支持简单查询，不支持嵌套。例如，**不支持**以下查询方式：

```plain
SQL

SELECT (
        SELECT accountName, 
               (SELECT contactName FROM contact) 
        FROM account
       )
FROM user
```
