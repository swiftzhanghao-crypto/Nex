向上查询指通过子到父的关系查询数据，用于多对一的关系中。

在向上查询时，可以在 SELECT、FROM 和 WHERE 子句中直接使用点号（.）表示关联对象上的字段（对象 API 名称.字段 API 名称）。

例如，查询联系人对象上满足以下条件的联系人并返回联系人的名称和客户名称：联系人对应客户的省份为“北京市”。

```plain
SQL

SELECT contactName, accountId.accountName
FROM contact
WHERE accountId.state = '北京市'
```

+ 向上查询时，如果子数据没有对应的父数据，则不会返回数据。
+ 向上查询最多支持 5 级，例如，select a.b.c.d.name from xxx。
+ 向上查询的字段数量（所有上级对象字段数的总和）最多支持 35 个。
+ 不支持查询多个关联关系字段（同时关联到相同对象）的场景。

例如：对象订单（_order）包含两个关联关系字段：创建人、审核人，两个字段都关联到对象用户（user）上，查询订单时不能同时查询创建人和审核人字段。

+ entityType（业务类型）字段不支持向上查询，例如不支持 entityType.typeName。
