+ 询结果为 JSON 串。
+ 有些情况下，需要手动转换字段类型。

XOQL 的查询结果为字符串（String）类型，需要根据实际字段类型进行转换。例如，查询结果返回的字段“公司人数”的值是“1230”，而字段的实际类型为整数类型（int），此时需要手动转换成整数类型（int）。

+ 查询结果中的 value 不会出现为 NULL 的情况。

XOQL 查询结果为 JSON 串，在 JSON 串中，不会出现 key 有值而 value 为 NULL 的情况。

+ 如果没有使用 ORDER BY 子句，则不能保证查询结果的记录顺序。
+ 一个对象的最大返回记录数为 3000 条。父记录和子记录的总记录返回数不超过 3000 条。例如，在进行向下查询时，共查询到了 3000 条记录，且未在子查询中使用 LIMIT 子句对记录数进行限制。此时，如果第 1 条父记录包含 3000 条子记录，则第 2 条之后的所有父记录将不会再包含子记录。
