## 多态字段作为返回字段
查询返回多态字段时，其返回的字段值包含两项：对象 ID、数据 ID。

例如，查询标准对象活动记录上的“来自”字段：

```plain
SQL

SELECT id,activityRecordFrom_compound FROM activityrecord
```

返回值 API 名称说明：

+ activityRecordFrom_compound：“来自”字段的 API 名称
+ activityRecordFrom：对象 ID 的 API 名称
+ activityRecordFrom_data：数据 ID 的 API 名称

其他多态字段的对象 ID 和数据 ID 的 API 名称与“来自”字段的名称规则类似。例如，标准对象服务费用明细上的“目标对象”字段，其字段 API 名称为 targetObject_compound，对象 ID 的 API 名称为 targetObject，数据 ID 的 API 名称为 targetObject_data。

## 多态字段作为查询条件
在查询条件中不能直接使用多态字段，需要使用对象 ID 或 数据 ID 进行筛选。

例如：

查询多态关联对象是“客户”的活动记录：

```plain
SQL

SELECT id,activityRecordFrom_compound FROM activityrecord WHERE activityRecordFrom = '1'
```

查询多态关联对象是“销售机会”且关联数据是“仁科互动销售机会”这条数据的活动记录：

```plain
SQL

SELECT id,activityRecordFrom_compound FROM activityrecord WHERE activityRecordFrom = '3' AND activityRecordFrom_data = '3335507670711919'
```
